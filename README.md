# 北京林业大学/强智教务系统自动评教

一个用于强智教务系统学生评教页面的油猴脚本。进入学生评教课程列表页后，点击右上角“开始自动评教”，脚本会自动读取未完成的评教入口、填写表单并逐项提交。

本脚本默认适配北京林业大学教务系统。北林教务系统底层采用强智教务系统，因此其他使用强智教务系统的学校通常只需要修改脚本头部的匹配网址即可使用。

> 提交评教后通常不能修改。请在使用前确认脚本配置符合你的真实评价意愿，并自行承担使用后果。

## 功能

- 在评教课程列表页注入“自动评教”操作面板
- 自动识别列表中的“评教”入口
- 后台逐项读取评教表单，避免浏览器弹窗拦截
- 自动填写教师评价、学生自评和个性化标签
- 自动提交并在页面日志中显示成功或失败项
- 支持在单个评教页面直接自动填写并提交当前页
- 提供 `dryRun` 调试模式，可只生成数据不提交

**如果你访问不了Greasy Fork，可尝试手动安装：**

1. 安装用户脚本管理器，例如 Tampermonkey、Violentmonkey 或 ScriptCat。
2. 打开脚本文件：[自动评教.user.js](./自动评教.user.js)。
3. 在脚本管理器中新建脚本，粘贴文件内容并保存。
4. 登录教务系统，进入教学评价/学生评教课程列表页。
5. 点击页面右上角“开始自动评教”。

## 配置

脚本顶部有 `CONFIG` 配置区：

```js
const CONFIG = {
  teacherOptionIndex: 2,
  selfOptionIndex: 2,
  reasonText: "课程内容清晰，老师讲解认真负责。",
  preferredTags: ["严师益友", "学者风范", "言传身教", "耐心细致", "条理清晰", "善于引导学生"],
  maxTags: 6,
  delayMs: 900,
  dryRun: false,
};
```

常用调整：

- `teacherOptionIndex`: 教师评价选项序号。`1` 通常是“非常满意”，`2` 通常是“满意”。
- `selfOptionIndex`: 学生自评选项序号。
- `reasonText`: 评价理由文本。建议保持 10 到 50 个汉字左右。
- `preferredTags`: 优先选择的个性化标签文字。
- `maxTags`: 最多选择几个标签，当前系统限制为 6。
- `delayMs`: 每门课程提交后的等待时间。
- `dryRun`: 改为 `true` 后只填充和生成提交数据，不真正提交，适合调试。

## 改为强智教务系统通用脚本

如果你的学校也使用强智教务系统，但域名不是 `newjwxt.bjfu.edu.cn`，通常只需要修改脚本顶部的 `@match` 网址。

例如脚本默认是：

```js
// @match        http://newjwxt.bjfu.edu.cn/jsxsd/xspj/*
// @match        https://newjwxt.bjfu.edu.cn/jsxsd/xspj/*
```

假设你的教务系统地址是 `jwxt.example.edu.cn`，可改为：

```js
// @match        http://jwxt.example.edu.cn/jsxsd/xspj/*
// @match        https://jwxt.example.edu.cn/jsxsd/xspj/*
```

如果学校教务系统把强智系统部署在不同路径，也需要同步修改路径部分。关键是让脚本能够匹配到学生评教列表页和评教表单页，一般评教相关路径包含 `/jsxsd/xspj/`。

修改后进入本校“教学评价/学生评教”页面测试。建议先把 `dryRun` 改为 `true` 做一次调试，确认脚本能识别评教入口后，再改回 `false`。

## 使用说明

1. 登录 `newjwxt.bjfu.edu.cn`。
2. 进入“教学评价”中的学生评教课程列表页。
3. 确认课程列表中有“评教”入口。
4. 点击右上角“开始自动评教”。
5. 等待脚本日志显示完成。
6. 刷新页面，确认“是否提交”状态。

## 隐私

脚本默认只在 `newjwxt.bjfu.edu.cn/jsxsd/xspj/*` 页面运行，不引入第三方脚本，不向第三方服务器发送数据。改为其他强智教务系统域名后，脚本会在对应教务系统会话内读取评教表单并向该系统原提交接口提交表单。

## 注意事项

- 本项目仅用于减少重复点击和填写操作，不代表对课程或教师的真实评价建议。
- 请根据个人真实意愿修改选项和标签后再使用。
- 教务系统页面结构变化后，脚本可能失效。
- 如果日志提示未登录或返回页面不是评教表单，请重新登录教务系统后再试。
- 发布到 GitHub 时不要提交本地保存的评教 HTML 页面，它们可能包含个人信息或课程信息。
